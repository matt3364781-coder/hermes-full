# Swing Point 区域标签 — Kimi 方案（2026-05-24）

## 核心问题

百分位标签（P5/P95）本质是均值回归信号，不是结构性支撑/阻力。
Swing Point 单点标签 + LogisticRegression = 样本稀疏（~2%）→ 维度灾难 → recall < 10%。

## 解法：把「点」扩展成「区域」

波峰/波谷检测后，向两侧扩展 zone_radius 根 K 线 → 从 ~2% 拉到 ~15-20% 样本量。

## 参数解释

- `left=5, right=5` — 左右各 5 根 K 线的极值检测窗口
  - 日线上 = 各约 1 周，共 2 周窗口找极值
  - 5min 上 = 各 25 分钟
- `zone_radius=3` — 极值前后各 3 根 K 线都标为同一区域
  - 日线上 = 约 1 周的区域（共 7 根）
  - 5min 上 = 15 分钟的区域

## 标签分配逻辑

| 条件 | 标签 | 含义 |
|------|------|------|
| 在波谷区域内 | 0 | 支撑区域 → CALL |
| 在波峰区域内 | 2 | 阻力区域 → PUT |
| 非区域 + close > 最近波峰 | 3 | 突破前高 → CALL |
| 非区域 + close < 最近波谷 | 4 | 跌破前低 → PUT |
| 其余 | 1 | 中性 |

## 与百分位标签的关键区别

| 方面 | 百分位 (旧) | Swing Point 区域 (新) |
|------|------------|---------------------|
| 结构含义 | 均值回归（Bollinger类） | 局部极值结构位 |
| 样本分布 | 0/2=5% 各，1=90% | 0/2=15-20% 各 |
| 突破标签 | 3/4=P1/P99（极罕见） | 3/4=清晰突破前高/跌破前低 |
| LR 友好度 | ❌ 中性类完全主导 | ✅ 支撑/阻力区样本够 |
| 狗基线 | 狗猜中性=90% | 狗猜中性=~60% |

## 训练超参数

执行 `train_daily_sp.py` 的 3-Fold × 4C × 2balanced 搜索（24 轮训练）。

在 engine.py 中的函数签名：
```python
from kronos.engine import detect_range
labels = detect_range(df['Close'], df['High'], df['Low'],
                       left=5, right=5, zone_radius=3)
```
