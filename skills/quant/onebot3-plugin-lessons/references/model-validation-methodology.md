# 量化模型驗證方法論（2026-05-24 定稿）

本文件濃縮這輪 KronosCluster v1.2 開發中所有驗證相關教訓。

## 0. 鐵律：永遠不要直接報默認參數的訓練準確率

C=1.0（sklearn 默認）在 832 維 × ~400 樣本下報了 97.9% train / 74.6% test。
用戶看到 97.9% 直接暴怒：「太離譜了大哥你能不能好好檢查啊？？？」

**教訓：** 先掃 C 值、先對比狗基線、先用時間序分割 → 再報數字。
寧說「待確認」也不報 97.9% 這種明顯有問題的數字。

## 1. C 值掃描（強制 — 禁止默認 C=1.0）

832 維 × ~400 訓練樣本 = 維度災難。默認 C=1.0 過擬合 ~28pp。

```python
for C in [0.01, 0.05, 0.1, 0.5, 1.0]:
    clf = LogisticRegression(C=C, solver='lbfgs', max_iter=2000, random_state=42)
    # 3-fold 取平均
```

選擇標準：最大 3-fold 平均 test acc。

## 2. 狗基線對比（強制 — 必須同步報告）

永遠預測「中性區（class 1）」叫狗策略。對比：
- 模型比狗高 <10pp → 不可用
- 日線百分位模型：75.2% vs 狗 ~54%（+21pp ✅）
- Swing Point 區域模型：44.3% vs 狗 ~20%（+24pp ✅ — 但絕對值太低）

## 3. 時間序驗證（強制 — 禁止 shuffle）

Shuffle 有 ~33pp 水分（55d 模型 shuffle 70.5% → 真實 OOS 37.3%）。

```python
# 3-fold Forward Chaining
folds = [
    ("早期", 0, n*0.50, n*0.50, n*0.70),
    ("中期", 0, n*0.70, n*0.70, n*0.85),
    ("晚期", 0, n*0.85, n*0.85, n),
]
```
報告用 3-fold 平均，不報最大值。

## 4. 標籤與特徵對齊檢查

Kronos hidden state 是**時序壓縮向量**，不是**幾何特徵向量**。

| 標籤方法 | 對齊 Kronos？ | 結果 |
|----------|--------------|------|
| 百分位 P5/P95（均值回歸） | ✅ 趨勢編碼天然對齊 | 75.2% |
| Swing Point 區域（曲率） | ❌ 需要局部曲率特徵 | 44.3% |

**核心原則：** 標籤方法必須與特徵幾何對齊，否則模型不可能學好。

## 5. class_weight 使用規則

| 標籤類型 | class_weight | 原因 |
|----------|-------------|------|
| 百分位（~5%/90%/5%） | ❌ 不加 | 分布天然均衡，balanced 壓中性區權重倒扣 |
| Swing Point（~15%/70%/15%） | ✅ 加 balanced | 稀有類需要加權 |

## 6. 5min 模型死刑判決

758 樣本 × 832 維 × 5 類 → 每類平均 150 樣本擬合 832 維超平面。
支撐 recall 7% = 數學上不可救。

復活條件：>7,500 樣本（stride=5）+ 窗口放大到 left=right=20（100 分鐘結構）。

## 7. 報告格式

```
模型: LogisticRegression C=0.01
數據: SPY daily 2022~now (1100 bars, stride=2 → 479 samples)
驗證: 3-fold Forward Chaining
Fold 1: 85.1% (狗 63.8%)
Fold 2: 65.7% (狗 45.7%)
Fold 3: 74.6% (狗 53.5%)
平均:  75.2% (狗 ~54%, +21pp)
過擬合: 8.3pp ✅
```
