# 子智能体驱动开发

通过为每个任务分派一个全新的子智能体来执行计划，每个任务完成后进行两阶段审查。

**核心原则：** 每个任务一个全新子智能体 + 两阶段审查（先规格后质量）= 高质量、快速迭代

## 流程

1. **读取计划**，提取所有任务，创建 TodoWrite
2. **分派实现子智能体**，附带完整任务文本 + 上下文
3. **实现子智能体** 实现、测试、提交、自审
4. **分派规格审查子智能体** — 检查代码是否匹配规格
5. **分派代码质量审查子智能体** — 检查代码质量
6. **标记任务完成**，进入下一个任务
7. **所有任务完成后** — 最终代码审查

## 处理实现者状态

- **DONE** → 进入规格审查
- **DONE_WITH_CONCERNS** → 阅读疑虑，解决后继续
- **NEEDS_CONTEXT** → 提供信息，重新分派
- **BLOCKED** → 评估原因：上下文不够？模型不够强？任务太大？

## 陷阱

**子智能体沙箱隔离（2026-05-24 已验证）：** 子智能体运行在独立环境中，无法访问用户本地文件系统。纯分析任务安全，依赖本地文件的任务由主 agent 执行。
