# 路径幻觉与记忆污染 — 2026-05-17

## 场景

用户问："/home/ubuntu/onebot 这个目录还有东西吗"

## 错误

我回答："之前只在 /home/ubuntu/onebot 下找，没找到"——但这个目录从未存在过，我也从未在该目录执行过任何操作。

## 根因

记忆污染。我在记忆里记了 `/home/ubuntu/onebot` 这个路径，但从未验证。将记忆里的路径假设当作已验证的事实陈述，并用"之前..."包装成看似合理的解释。

## 纠正

- 禁止用"之前..."来描述未实际执行过的操作
- 只能陈述当前验证结果（"我执行了 find，结果是..."）
- 找不到文件时，正确话术："我找不到，让我全局搜索"
- 记忆里的路径 ≠ 已验证的路径

## 规则

1. **任何路径声称必须先 `ls` 或 `find` 验证**，禁止凭记忆断言文件系统状态
2. **当找不到文件时**，用 `find /home/ubuntu -name "*pattern*"` 全局搜索，而非假设路径
3. **"之前我在..."这种话术是幻觉的温床**——它暗示了执行历史，实际可能从未发生
4. **被用户追问"你干嘛会有这目录的幻觉"时**，承认是记忆污染，不要编造解释
5. **记忆删除请求要立即执行**，不要追问细节

## 记忆删除执行规范

- 用户要求删除记忆 → 立即执行，失败时说明原因
- 不要反复尝试不同关键词变体（显得笨拙且浪费时间）
- 如果记忆工具无法定位，直接说"我找不到这个记忆条目"
- 不要为删除失败编造解释

## 目录清理规范

- 用户说"删"=立即执行，零确认
- 但删除前快速检查内容，避免误删有用数据
- 当发现多个同名/相似目录时，用 `find` 列出全部，让用户确认哪个是主目录
- 清理后确认：只剩唯一权威路径
