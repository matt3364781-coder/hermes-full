# 隐性执行许可识别 — 2026-05-17

## 场景

用户展示了 technical 层逐年数据（2020-2025 命中率/宽度统计）后说："你必须要做好xg才行啊"

## 分析

这不是可行性咨询，而是**隐性执行指令**——用户在用数据说明现状，暗示需要改进。

## 语气区分规则

| 语气 | 示例 | 处理方式 |
|------|------|---------|
| **疑问语气** | "能不能..." / "可以吗" / "可行吗" | 只回答可行性+方案，等明确指令（"去做"/"开干"） |
| **要求语气** | "必须要..." / "得做好..." / "该优化了" | 隐性执行许可，可以开始 |
| **直接指令** | "去做" / "开干" / "执行" / "删" | 立即执行 |

## 关键区分点

- "能不能" → 等明确指令
- "必须要" / "得做好" → 可以开始，但快速确认"我开干了？"更安全
- 用户展示数据 + 抱怨现状 = 执行信号

## 教训

- 不要只识别显性的"去做"/"开干"
- 用户的语气、上下文、数据展示都是执行许可的信号
- 不确定时，快速确认比直接执行更安全
- 但也不要过度确认——用户说"必须要"时，确认一次即可
