# 编写技能

**编写技能就是将测试驱动开发应用于流程文档。**

先写测试（压力场景），看它失败（基线），编写技能（文档），看它通过，然后重构（堵漏洞）。

**核心原则：** 没有失败的测试就不写技能。

## SKILL.md 结构

**Frontmatter（YAML）：**
- 必需：`name` 和 `description`
- `name`：只使用字母、数字和连字符
- `description`：以"Use when..."开头，聚焦于触发条件，**不总结工作流**

## 铁律

```
没有失败的测试就不写技能
```

无例外。不适用于"简单的添加"。先写技能再测试？删掉它。

## 反模式

- 叙事式示例（"在 2025 年 10 月的会话中…"）
- 多语言稀释（同时在 3 种语言中实现同一示例）
- 流程图中的代码
